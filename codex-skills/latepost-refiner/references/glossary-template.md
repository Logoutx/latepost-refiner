# Glossary Template

Write this to `<out>/校对表.md`. It is a cumulative research artifact shared by all transcript refinements for the same topic/company.

```markdown
# <主题> 校对表

## 采访背景
- 时间 / 地点：
- 采访者：
- 受访者：
- 主题 / 领域：
- 涉及公司 / 机构：

## 人名（各种写法 -> 统一为 + 身份/依据）
| 转录里的各种写法 | 统一为 | 身份 / 当时 title | 真名核实依据 |
|---|---|---|---|
|  |  |  |  |

## 品牌 / 公司 / 产品（写法 -> 统一）
- 自家品牌 / 母公司：
- 同业 / 竞品：
- 上游 / 供应商 / 工厂：
- 平台 / 渠道：
- 投资方 / 机构：
- 系统 / 工具：
- 其它专有名词：

## 需特别修正 / 标注的转写错误
- 同音字错：
- 英文听写错：
- 时间戳：一律删除。
- 英文乱码：能判断就替换，判断不了就顺掉，不留乱码。
- `（音）`：能确定就改正去标；不能确定就保留。
- 事实存疑项：保留原文 + `（音，存疑）`，不臆改。

## 疑似同指，待确认
- （待补充）

## 未核实 / 待补查
- （待补充）

## 精校规范要点
1. 保持对话体，保留发言人标签。
2. 删口癖与重复，但不改语气风格与原意、不加观点。
3. 理顺可读性，保留有个性/有信息量的金句。
4. 按主题加不编号 `##` 小标题。
5. 严格按本表统一人名/品牌/术语。
6. 保留全部事实细节。
7. 规范发言人标注。
8. 抬头使用 `# 标题` + 第二行斜体采访说明。
9. 中文排版使用全角引号、阿拉伯数字、Pangu spacing。
10. 长文件分大块接力写到结尾。
```

Rules:
- Treat public-source conclusions as research notes, not just editing aids.
- Carry forward prior verified entities when extending an existing glossary.
- Do not re-verify prior confirmed entities unless the current batch creates a conflict.
- Keep unresolved ambiguities visible instead of guessing.
