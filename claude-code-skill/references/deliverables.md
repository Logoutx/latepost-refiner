# 时间线、访谈总结、逻辑顺序稿 的结构模板

仅当本次范围包含相应产出时才做。三者都基于**精校后的内容**；时间线还要对照**公开资料**；逻辑顺序稿只重排、不增删。

---

## A. 时间线（`<输出>/<主题>时间线.md`）

目的：把访谈口述与公开资料对照，排出公司/人物的发展线，并标清每条信息的来源可信度。

建议结构：

```markdown
# <主题> 时间线

> 整理依据：<时间><对象>访谈（采访：XXX），结合公开资料（来源…）交叉校对。
> 采访时点为 <X>，文中“采访时”均指此时；之后为公开资料补充。
> 标注：【公开】=公开资料可查；【访谈】=访谈口述；【公开+访谈】=两者互证。

## 一句话背景
<谁、做什么、关键打法、对标>

## 时间线
### <年份/阶段>：<小标题>
- **<年月>**【公开/访谈/公开+访谈】<事件，含具体数字/人物/金额>
- ...

## <采访之后>（公开资料补充，看后续走向）
- ...

## 关键人物对照表（含公开资料核实的真名）
| 访谈中称呼 | 真实姓名 | 角色（采访时） | 背景 / 公开信息 |
|---|---|---|---|
| ... | XXX（已核实）/ 真名未公开 | ... | ... |

> 真名核实说明：逐条标依据（哪篇报道/工商信息）；查不到的标“真名未公开”，并说明若要可通过工商数据库（员工持股平台合伙人名单）核对。存疑的投资主体/数据标注待考。
```

做法提示：
- 先从访谈里抽出所有带时间/阶段的事实（成立、产品、融资、人事、渠道、出海…）。
- 再用 WebSearch / WebFetch 按“公司名 + 融资/成立/创始人”核实年份、轮次、金额、投资方、实控人等，并标来源。
- 访谈与公开冲突时，两边都列、注明分歧，不强行二选一。

---

## B. 访谈总结（`<输出>/<主题>访谈总结.md`）

目的：把一套访谈提炼成可复用的研究笔记。三部分：要点 / 金句 / 洞察。

建议结构：

```markdown
# <主题> 访谈总结

> 基于 <时间> 对 <对象> 的访谈（采访：XXX）。受访者：A（职务）、B（职务）…
> 全文分三部分：分类要点、金句 Quotes、行业与公司/人物洞察。

## 分类要点
### <主题分类，如 创始人哲学 / 战略 / 产品 / 投放 / 供应链 / 组织>
- <要点，带关键事实>
### ...

## 金句 Quotes
**<人名（职务）>**
- “<忠实引用，保留语气；只清理口癖，不改原意>”
- ...

## 行业与公司/人物洞察
### 关于行业
- ...
### 关于这家公司 / 这个人
- ...（含看点与风险）

> 配套文档：同目录《<主题>时间线.md》、Transcripts/ 下 N 份精校访谈全文。
```

做法提示：
- 要点按“角色/主题”分类，方便检索；每条尽量带一个具体事实或数字支撑。
- 金句必须忠实引用（精校只去口癖、不改意思）；按发言人归类。
- 洞察分“行业层面”和“该公司/人物层面”，后者点出看点与风险，体现判断而非复述。

---

## C. 逻辑顺序稿（`<输出>/逻辑顺序/<标题>.md`）

目的：把**同一篇**访谈的问答从“录音顺序”重排成“叙事顺序”，让散落各处、其实同属一条线的问答聚成完整故事。**保序重排·原文照搬**——问答块整段照搬精校稿、一字不改一处不漏，只换位置；精校稿本身不动（它仍是忠实录音顺序的可引用存档）。

建议结构：

```markdown
# <英文名（中文名）title> · 逻辑顺序稿
*基于精校稿重排为叙事顺序，内容照搬未改，仅调顺序 + 少量 [编者] 衔接；原顺序见 Transcripts/<标题>.md*

## 主线脉络（导读）
<一段话：这次访谈的主线是什么、你按什么逻辑重排（时间/因果/主题块）>

## <线索1：自描述小标题，如 创业缘起>
（按时间）
<记者：照搬精校稿原文——发言人标签沿用精校稿的原样式（精校稿不加粗就别加粗）>
<受访者：照搬原文>
> [编者] <仅在被移动打断指代/承接时加一句衔接，如：此段原在访谈后段，承前文同属早期创业>
...

## <线索2：自描述小标题，如 战略转折>
（按 问题 → 洞察 → 决定 → 结果）
...
```

做法提示：
- 先理出 3–7 条叙事线索，各给一个**不编号**的自描述小标题；线索之间按自然脉络排（通常先缘起、再主题块）。
- 每条线索选一个内部顺序逻辑：讲历史按时间，讲决策按 因→果，讲产品/事件按 起因→经过→结果。
- 问答**整段照原文搬**，保留 `发言人：` 标签**及其在精校稿里的原样式**（精校稿是 `张三：` 就写 `张三：`，别改成 `**张三：**` 加粗或其它样式）与全部事实细节，只换位置；**绝不**改写措辞、补话、下结论。
- `[编者]` 衔接句要克制——只在移动真的打断了指代/承接时才加，且明确标 `[编者]`。
- **不丢不重**：精校稿每段实质问答都要出现且只出现一次；横跨两线索的放主线索一次、另一处用一句 `[编者]` 指路。
