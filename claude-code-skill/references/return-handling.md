# Workflow return handling (Claude Code fast path)

The `Workflow({ scriptPath, args })` dispatch returns a result object. Walk these fields, then go straight to **Step 5 (wrap-up)** in `SKILL.md`.

## вӣ” First check `auditUnavailable` вҖ” a failed run, not a successful one

If top-level **`auditUnavailable`** (`[{ path, label }]`) is **non-empty**, the deterministic audit **could not run** for those жҲҗзЁҝ even after one retry (e.g. the sandbox audit sub-agent kept returning unparseable output, or a sandbox error). **The run has FAILED.** The refined transcripts вҖ” and any жҖ»з»“/ж—¶й—ҙзәҝ/йҖ»иҫ‘зЁҝ derived from them вҖ” are **unaudited, therefore unverified, therefore not deliverable as-is**. Do **not** write a success summary. Instead: (1) state plainly that the audit did not run and the deliverables are unaudited/invalid, naming each file; (2) the files are still on disk (work preserved) вҖ” run the audit **by hand** for each: `node "<this skill dir>/audit_refined.mjs" --source "<src>" --refined "<out>"`, read its JSON, and only then decide whether each жҲҗзЁҝ (and its derivatives) can be trusted. "Audit unavailable" is never "audit passed"; a gate that is silently skipped is not a gate.

## If the run partially fails вҖ” do NOT re-refine

The pipeline is built so a **cheap gate agent** can never cost the expensive work: refine runs even when scout fails (it reads the source directly), and completeness is judged by the **deterministic source-aware audit** (no separate haiku check agent) вҖ” its `ending_missing` gate is what marks a file `incomplete`. But if the **harness itself** drops a long run mid-way (зҪ‘з»ңеҚЎжӯ»зӯү), the refined transcripts вҖ” and any deliverables вҖ” are **already on disk** under `<output>/Transcripts/` (and `йҖ»иҫ‘йЎәеәҸ/`, root). Recover cheaply, never re-refining:

- **Resume**: re-invoke `Workflow({ scriptPath, args, resumeFromRunId: '<the run id>' })` вҖ” completed refine agents return cached, only the failed cheap steps re-run.
- **Or finish by hand**: if the refined files exist, run **only** the missing pieces against the on-disk refine вҖ” the deterministic audit (`node "<this skill dir>/audit_refined.mjs" --source "<src>" --refined "<out>"`) for completeness, and one deliverable subagent each for any missing и®ҝи°ҲжҖ»з»“ / ж—¶й—ҙзәҝ / йҖ»иҫ‘йЎәеәҸзЁҝ. This is the "жҲ‘жүӢеҠЁиЎҘе®Ң check + дәӨд»ҳзү©гҖҒжІЎйҮҚи·‘зІҫж Ў" flow вҖ” that's the right move, not a full re-run.

1. **`glossary`** вҖ” `Write` it in full to `<output dir>/ж ЎеҜ№иЎЁ.md` to archive. It is **cumulative** (it already incorporates any `priorGlossaryText` you passed), so writing it back simply supersedes the old file вҖ” no manual merge. Before writing, glance over the typesetting: verification sources, same-referent reasons, and other full-sentence Chinese notes should use Arabic numerals, Pangu spacing, and full-width curly quotes. Subagents mostly get this right; fix any stragglers.
2. **`failed`** вҖ” these files weren't produced. Do them by hand per the manual pipeline (`references/manual-steps.md`, Steps 1вҖ“2).
3. **`incomplete`** вҖ” the deterministic audit's `ending_missing` gate fired: the source's last sentence never surfaced in the жҲҗзЁҝ, so the output didn't reach the ending. Each item is `{ path, note }`. Dispatch a refine subagent to continue to the end.
4. **`unchecked`** вҖ” completeness could **not** be determined: no source-aware audit ran for the file (the audit capability was absent, or the sandbox audit sub-agent didn't return parseable JSON). This is not the same as `incomplete` вҖ” don't treat the file as passed *or* as truncated; self-check against the source ending (run `audit_refined.mjs` by hand, or eyeball with `tail`).
4a. **`audit` (per refined entry) + `auditFailed` (top-level, В§2)** вҖ” the source-aware audit now runs **inside** the pipeline, per file, right after refine/stitch and before logic/жҖ»з»“/ж—¶й—ҙзәҝ (no longer only a manual Step-5 pass). Each `refined[i].audit` is `{ status, hardFindings, softFindings, repaired, anchorsAdded, auditUnavailable }`. A **hard** finding (`content_gap` вҖ” a source stretch that never surfaced in the жҲҗзЁҝ with no fold trace, the silent-omission failure; or `quote_style` вҖ” ASCII зӣҙеј•еҸ·зҙ§иҙҙдёӯж–Ү / гҖҢгҖҚгҖҺгҖҸ) triggers **one** automatic repair attempt (a refine sub-agent patches only the named spots) + **one** re-audit. If it passes, `repaired: true` and you're done. If it's **still hard**, the file is listed in top-level **`auditFailed`** (`[{ path, findings }]`) and a visible `> вҡ гҖҗеҶ…е®№зјәеҸЈпјҡжәҗж–Үд»¶з¬¬ X-Y иЎҢвҖҰгҖ‘` marker is dropped into the жҲҗзЁҝ. For each `auditFailed` file: (1) tell the user which line ranges are missing and why (often a content-policy pass on a sensitive segment); (2) offer to re-refine **just those source line ranges** вҖ” on a different provider if it looks policy-driven вҖ” and hand-merge; never re-refine the whole file. Note: жҖ»з»“/ж—¶й—ҙзәҝ/йҖ»иҫ‘зЁҝ were generated after the gate, so they inherit the marked gap вҖ” regenerate them once the content is backfilled. `auditUnavailable: true` means the audit couldn't run (e.g. the sandbox sub-agent didn't return parseable JSON) вҖ” treat the file as unverified and run `audit_refined.mjs` by hand. Model-inserted `жңӘзІҫж Ўж®ө` markers (RULES 14 last-resort placeholders) surface as `model_marker` findings; the audit's line numbers are authoritative, not the marker's.
4b. **Source anchors** вҖ” after the audit (and any gap annotation), run `вҖҰ --anchors` per file: each `##` section gets `<!-- жәҗ LвҖҰ-LвҖҰ В· ж—¶й—ҙ-ж—¶й—ҙ -->` under its heading (invisible when rendered). Provenance for the desk: quote вҶ’ source lines вҶ’ recording timestamp. Only the жҲҗзЁҝ carries anchors вҖ” жҖ»з»“/ж—¶й—ҙзәҝ/йҖ»иҫ‘зЁҝ cite е°Ҹж Үйўҳ, inheriting traceability through them. Sections with no matched evidence are skipped by design.
5. **`scoutSuspect` / `scoutFailed`** (non-empty) вҖ” scouting was unreliable for these files: `scoutSuspect` = returned garbled content (still broken after a retry, usually a network-corrupted stream); `scoutFailed` = returned **nothing at all** (stalled, e.g. on an oversized merged file). **Either way the outputs are fine** вҖ” refine reads the source directly and ran regardless вҖ” but those files' glossary entries are missing/untrustworthy. Tell the user and suggest re-running scouting on just those files once the network is stable (or after splitting an oversized file).
6. **`headingConflicts`** вҖ” these files already had sub-headings but ran under 'none'. Fold into the Step 5 questions (keep / redo), re-running that file alone if needed.
7. **`suspectedDuplicates`** вҖ” groups written differently but suspected of being the same referent that the script did not auto-merge (e.g. зҺӢеӢҮ / жұӘеӢҮ). Already folded into `openQuestions`; ask the user in Step 5 whether they're the same.
8. **`networkUnverified`** (non-empty) вҖ” items the verify agent **skipped via the circuit breaker, never actually verified** (as opposed to "checked but not found"). In the Step 5 batch, offer a **re-verify option** (вҖңзҪ‘з»ңжҒўеӨҚдәҶеҗ—пјҹиҰҒдёҚиҰҒжҠҠиҝҷ N йЎ№иЎҘжҹҘдёҖйҒҚвҖқ). On confirmation, dispatch **one** sonnet subagent to verify just those via WebSearch вҖ” same discipline: only resolved with cited evidence, never fabricate, stop on consecutive errors вҖ” and append the conclusions to a гҖҢ## иЎҘж ёз»“и®әгҖҚ section at the end of `<output dir>/ж ЎеҜ№иЎЁ.md`. If a re-verify **corrects a name/term spelling already used in the outputs**, tell the user and propose one targeted replacement. Names follow the same strong-name guard: when a re-verify conclusion conflicts with an output's strong name, only report it, don't silently change it.
9. **`openQuestions`** вҖ” fold into the single Step 5 batch. Now also carries **з–‘дјјиҪ¬еҪ•иҜҜеҶҷгҖҒжңӘж ёе®һ** items: entities the scout flagged as likely ASR mishears (`suspect_asr`) that verify couldn't resolve вҖ” the class where a consistently mis-transcribed name has no variants and would otherwise slip the verify filter. The glossary marks each with вҡ ; surface them so the user confirms the correct еҶҷжі• (and if confirmed, propose a targeted replacement in any output that used the wrong spelling).
10. **`logic`** (the logical-order rewrites) вҖ” already written by the workflow to `<output>/йҖ»иҫ‘йЎәеәҸ/`. If any item's `missingSections` is non-empty, the per-section coverage check suspects that section was dropped from the rewrite; spot-check it and re-run that file alone if needed.
11. **`summary` / `timeline`** вҖ” already written to disk by the workflow.
12. **`crossFileConflicts` (M8 и·Ёж–Үд»¶дә’иҜҒ) вҖ” universal (CLI/web) edition only; this Claude Code edition does NOT produce it.** The batch-level cross-file numeric-consistency check (same entity + unit stated with conflicting values across two files, where each file is internally faithful to its own source) needs to read every refined file back plus the ж ЎеҜ№иЎЁ вҖ” which the Workflow sandbox cannot do (no fs; refined text never enters the orchestration layer), so it runs in the CLI/web layer instead.
